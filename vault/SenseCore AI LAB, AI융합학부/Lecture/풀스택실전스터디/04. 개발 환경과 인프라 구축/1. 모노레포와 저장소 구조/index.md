# 1. 모노레포와 저장소 구조
이 페이지에서는 프론트, 백엔드, 문서, 스크립트를 어떤 구조로 둘지 정합니다. 저장소 구조는 이후 개발 속도와 이해도를 좌우합니다.

# 이 페이지의 목표
- 모노레포 디렉터리 구조를 정한다
- 앱과 패키지 경계를 나눈다
- 문서와 실행 스크립트 위치를 정한다

# 산출물
- repo tree 초안
- 패키지 책임 표
- 스크립트 위치 규칙

# 체크리스트
- 모노레포 디렉터리 구조를 정한다
- 앱과 패키지 경계를 나눈다
