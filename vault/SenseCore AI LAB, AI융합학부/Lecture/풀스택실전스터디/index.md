# 풀스택실전스터디
이 책은 하나의 DDD 기반 이커머스 서비스를 처음부터 끝까지 만드는 실전 교본입니다. 서비스 기획에서 출발해 도메인 모델링과 화면 기획을 하고, 개발 환경을 세운 뒤 백엔드와 프론트를 함께 구현하며, 운영자 기능과 AI 상담 챗봇까지 붙인 다음, 마지막에 검증과 배포와 확장까지 닫습니다. 목표는 기술 조각을 따로 배우는 것이 아니라, 자연스러운 제품 개발 순서를 그대로 따라가며 하나의 풀스택 서비스를 완성하는 것입니다.

# 이 책의 목표
- 기획, 모델링, 화면 설계, 구현, 배포를 하나의 흐름으로 설명할 수 있게 합니다.
- 이커머스 도메인을 DDD 관점으로 자르고 경계를 유지하면서 구현할 수 있게 합니다.
- 백엔드와 프론트를 기능 단위로 함께 완성하는 실전 감각을 갖게 합니다.
- 운영자 기능과 AI 상담 챗봇을 서비스 본체에 연결할 수 있게 합니다.
- 검증, 운영, 확장까지 포함한 제품 개발 습관을 갖게 합니다.

# 이 책을 읽는 방법
- 처음부터 끝까지 순서대로 읽습니다.
- 각 단계 안의 하위 페이지를 모두 소화한 뒤 다음 단계로 넘어갑니다.
- 항상 현재 단계가 제품 전체에서 어떤 책임을 맡는지 먼저 확인합니다.
- 구현 전에는 반드시 기획과 모델링 결과를 먼저 고정합니다.

# 이 책의 읽는 순서
1. 서비스 기획과 요구사항 정의
2. DDD 모델링과 도메인 설계
3. 화면 기획과 사용자 경험 설계
4. 개발 환경과 인프라 구축
5. 제품 골격 구현: 백엔드와 프론트 시작
6. 카탈로그와 인증 풀스택 개발
7. 장바구니, 주문, 결제 풀스택 개발
8. 운영자 기능과 AI 상담 챗봇 개발
9. 통합 검증과 품질 고도화
10. 배포, 운영, 확장

이 순서는 기획이 먼저이고 구현이 나중이며, 프론트와 백엔드는 기능 단위로 함께 완성하는 순서입니다. 그래서 학생은 지금 만드는 코드가 제품 개발의 몇 번째 단계인지 잃지 않고 따라갈 수 있습니다.

# 각 단계에서 무엇을 배우는가
## 1단계. 서비스 기획과 요구사항 정의
- 어떤 문제를 해결하는 서비스인지 정하고 MVP 범위를 고정합니다.
- 사용자와 운영자와 AI 상담 흐름의 핵심 시나리오를 정리합니다.
## 2단계. DDD 모델링과 도메인 설계
- 유비쿼터스 언어, 바운디드 컨텍스트, aggregate, 서비스 경계를 설계합니다.
## 3단계. 화면 기획과 사용자 경험 설계
- storefront, admin, AI 챗 UI의 화면 구조와 사용자 흐름을 설계합니다.
## 4단계. 개발 환경과 인프라 구축
- 모노레포, compose, env, DB, CI 바닥을 세워 팀이 같은 방식으로 개발하게 합니다.
## 5단계. 제품 골격 구현: 백엔드와 프론트 시작
- 서버와 클라이언트의 실행 골격, 공통 계약, 첫 통합 실행을 만듭니다.
## 6단계. 카탈로그와 인증 풀스택 개발
- 상품 조회와 인증을 백엔드와 프론트에서 함께 완성합니다.
## 7단계. 장바구니, 주문, 결제 풀스택 개발
- cart, checkout, order, payment를 실제 서비스 흐름으로 연결합니다.
## 8단계. 운영자 기능과 AI 상담 챗봇 개발
- 운영자 화면과 AI 상담 기능을 서비스 본체와 연결합니다.
## 9단계. 통합 검증과 품질 고도화
- E2E, 성능, 보안, 운영 점검으로 제품 품질을 올립니다.
## 10단계. 배포, 운영, 확장
- 배포 경로를 닫고 운영 런북과 다음 확장 로드맵까지 정리합니다.
