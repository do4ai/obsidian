# 3. Aggregate, Entity, Value Object 설계
이 페이지에서는 상태가 중요한 개념들을 aggregate 단위로 정리합니다. cart와 order, payment의 경계도 여기서 선명해져야 합니다.

# 이 페이지의 목표
- aggregate 후보를 고른다
- entity와 value object를 구분한다
- 상태 전이의 중심 대상을 정한다

# 산출물
- aggregate 목록
- entity/value object 메모
- 상태 전이 초안

# 체크리스트
