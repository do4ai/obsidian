# 02. DDD 모델링과 도메인 설계
이 단계에서는 기획 결과를 도메인 언어로 바꿉니다. 이커머스의 핵심 개념을 어떤 경계로 자를지, 어떤 상태 전이를 중심으로 구현할지 정하는 단계입니다.

# 이 단계에서 배우는 것
- 유비쿼터스 언어와 이벤트 스토밍으로 핵심 흐름을 정리합니다.
- 바운디드 컨텍스트와 aggregate를 설계합니다.
- 도메인 서비스와 애플리케이션 서비스 경계를 고정합니다.
