# 01. 서비스 기획과 요구사항 정의
이 단계에서는 무엇을 만들지 먼저 고정합니다. 이커머스 서비스의 문제 정의, 사용자, MVP 범위가 흔들리면 뒤의 DDD와 화면 설계와 구현 순서가 모두 흔들립니다.

# 이 단계에서 배우는 것
- 서비스의 문제와 성공 기준을 정합니다.
- 사용자와 액터의 핵심 시나리오를 정리합니다.
- MVP 범위와 산출물 기준을 고정합니다.
