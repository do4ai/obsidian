이 해설 페이지는 이미지 VLM이 시스템 문제로 넓어질 때 어떤 추가 난제가 생기는지 점검하는 페이지입니다.

# 이 해설 페이지의 역할
- temporal modeling과 spoken dialogue 확장을 다시 정리하게 합니다.
- multimodal RAG와 tool use가 무엇을 바꾸는지 설명하게 합니다.
- serving과 latency가 왜 아키텍처 선택을 바꾸는지 복습하게 합니다.

# 문제를 풀 때 확인할 것
- 비디오 토큰이 이미지 토큰과 어떻게 다른지 설명할 수 있는가
- RAG와 agent loop가 붙으면 오류가 어떻게 복합화되는지 말할 수 있는가
- cost와 latency 제약이 실제 설계에 어떤 영향을 주는지 정리할 수 있는가

# 연결 복습
- 이 단계가 확실해야 8단계 frontier 연구를 “시스템까지 포함한 멀티모달” 관점에서 읽을 수 있습니다.
